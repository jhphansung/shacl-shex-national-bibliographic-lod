# SHACL과 ShEx를 활용한 국가서지 LOD의 품질 평가

본 저장소는 박진호(2026)의 학술 논문 「SHACL과 ShEx를 활용한 국가서지 LOD의 품질 평가: 온톨로지 설계 검증과 실데이터 검증의 이중 층위 분석」의 재현성 확보를 위한 부속 자료를 제공합니다.

## 연구 개요

본 연구는 RDF 검증을 위한 W3C 표준 언어인 SHACL(Shapes Constraint Language)과 ShEx(Shape Expressions)를 활용하여 국립중앙도서관 국가서지 LOD의 품질을 온톨로지 설계 층위와 실데이터 층위의 양면에서 진단하였습니다. 표본 추출이 아닌 전수조사 방식을 채택하여 2026년 4월 1일자로 공개된 약 9억 1,500만 트리플 전체를 분석하였습니다.

### 주요 발견

- **설계 층위**: 전체 속성의 41.4%가 정의역을 명시하지 않는 명세 불완전성, BIBFRAME 어휘의 미반영
- **실데이터 층위 (SHACL)**: 전체 적합률 99.73%, 위반의 99.6%가 서지(온라인자료)의 표제 누락에 집중
- **실데이터 층위 (ShEx)**: 연역 형상과 귀납 형상의 평균 미예상 속성 비율 98.91%
- **이중 층위 통합**: 설계에 부재하던 BIBFRAME 어휘가 실데이터에서는 광범위하게 사용되는 설계-실데이터 비대칭 확인

## 데이터 출처

본 연구의 분석 대상은 국립중앙도서관 국가서지 LOD(https://lod.nl.go.kr) 의 2026년 4월 1일자 벌크 데이터입니다. 데이터 자체는 약 48.5GB 규모로 본 저장소에는 포함되지 않으며, 위 URL의 공공데이터 페이지에서 직접 내려받으실 수 있습니다.

분석 대상 데이터셋은 다음 여섯 가지입니다.

| 데이터셋 | 폴더명 |
|---|---|
| 도서관 정보 | 기타도서관정보Library_json_20260401 |
| 서지(오프라인자료) | 서지데이터오프라인자료Offline_json_20260401 |
| 서지(온라인자료) | 서지데이터온라인자료Online_json_20260401 |
| 전거(개인명) | 전거데이터개인명Person_json_20260401 |
| 전거(단체명) | 전거데이터단체명Organization_json_20260401 |
| 전거(주제명) | 전거데이터주제명Concept_json_20260401 |

## 실행 환경

- **Python**: 3.13 (Anaconda 환경 권장)
- **운영체제**: 본 연구는 Windows 환경에서 검증되었으며, Linux/macOS에서도 동작합니다
- **필수 라이브러리**: `rdflib`, `pyshacl`

### 설치

명령 프롬프트(또는 터미널)에서 다음 명령으로 필요 라이브러리를 설치합니다.

```bash
python -m pip install rdflib pyshacl
```

## 실행 순서

각 스크립트 상단의 `DATA_ROOT` 변수를 본인 환경의 데이터 폴더 경로로 수정한 뒤 실행하십시오. 예를 들어 Windows에서 `D:\데이터` 폴더에 데이터셋을 두었다면 다음과 같이 설정합니다.

```python
DATA_ROOT = r"D:\데이터"
```

### 1단계: 데이터 규모 진단 (논문 6.1절)

```bash
python scripts/01_데이터규모진단.py
```

6개 데이터셋의 파일 수, 용량, 레코드 수, 클래스별 인스턴스 수, 속성별 출현 빈도를 전수 집계합니다.

### 2단계: SHACL 검증 (논문 6.4절)

```bash
python scripts/02_SHACL검증.py
```

클래스별 NodeShape 기반 SHACL 전수 검증을 수행합니다. RDFS 추론을 활성화하며, 표기 분산 속성은 `sh:alternativePath`로 통합 처리합니다. 대규모 데이터 처리를 위해 파일 단위 스트리밍 방식으로 검증하며 메모리 부담을 최소화합니다.

### 3단계: ShEx 검증 (논문 6.5절)

```bash
python scripts/03_ShEx검증.py
```

실데이터로부터 귀납적으로 ShEx 형상을 추출하고, 5장의 연역 형상과 비교하여 속성 일치율, 카디널리티 일치율, 미예상 속성 비율을 산출합니다. 속성 채택 임계값은 0.1을 적용합니다(즉 클래스 인스턴스의 10% 이상에서 출현하는 속성만 형상에 포함).

## 방법론적 참고 사항

- **귀납 ShEx 형상 추출**: 본 연구는 sheXer 라이브러리의 Windows 환경 설치 제약(Rust/C++ 빌드 도구 요구)을 우회하기 위해 rdflib 기반으로 클래스별 속성 출현율과 카디널리티를 직접 집계하는 방식을 채택하였습니다. 알고리즘이 명시적으로 기술되어 재현성과 방법론적 투명성이 강화되었습니다.
- **전수조사 방식**: 표본 추출에 따른 대표성 한계를 피하기 위해 약 9억 1,500만 트리플 전체를 결락 없이 분석하였습니다. SHACL 검증은 약 17시간(컨슈머 데스크톱 환경 기준) 소요됩니다.
- **카디널리티 해석 시 유의점**: JSON-LD 직렬화에서 단일 값이 배열로 직렬화되는 특성으로 인해 귀납 형상의 카디널리티가 복수허용으로 산정되는 경향이 있습니다. 본 연구의 카디널리티 일치율 지표(1.79%)는 데이터 품질보다 직렬화와 추론 알고리즘의 상호작용을 반영하는 측면이 있음을 유의하시기 바랍니다.
